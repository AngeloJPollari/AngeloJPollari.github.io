Project Overview

This project simulates a small enterprise network with a headquarters site, a remote site, and an internet-facing perimeter. I built a layered architecture that includes:

  • Internal LAN with multiple switches and VLAN-aware interfaces
  • A routed core with a main router and a remote router connected via IPsec VPN
  • An ASA firewall segmenting inside, DMZ, and outside networks
  • Central services (DHCP, NTP, syslog) for consistent configuration and log correlation
  • AAA using TACACS+ and RADIUS for device administration
  • A wireless LAN controller with lightweight access points providing separate staff and guest Wi-Fi

The objective was to design, configure, and validate a secure end-to-end network that demonstrates defense-in-depth across routing, authentication, perimeter security, monitoring, and wireless access.

Architecture & Security Components

4.1 Core LAN & VLAN Segmentation

Tools Used: Cisco 3650 Switches, Cisco 4331 Router

  • Configured Layer 2 switching with two core switches interconnected via proper crossover cabling
  • Assigned VLAN 1 management interfaces on switches with static IPs to allow remote management
  • Set default gateways on switches and PCs pointing to the main router, enabling inter-network routing
  • Verified connectivity using ICMP (ping) and inspected MAC/ARP tables to confirm proper learning and forwarding

4.2 AAA, TACACS+, RADIUS & Device Access

Tools Used: AAA Server, Cisco 4331 Routers, Cisco 3650 Switches

Deployed a dedicated AAA server to centralize authentication for routers and switches.

Configuration:

  • Configured TACACS+ for router R1 and one switch, and RADIUS for another switch to compare behavior
  • Applied aaa new-model with login authentication via the AAA server (with local backup)
  • Enable/privileged mode protected by AAA
  • Created multiple user accounts on the AAA server

Testing:

  • Tested authentication via Telnet from PCs to switches/routers
  • Confirmed fallback to local credentials when the AAA server is unavailable

Note: This project uses centralized AAA for admin logins; it does not implement full multi-factor authentication (2FA) in this lab environment.

4.3 Site-to-Site IPsec VPN

Tools Used: Cisco 4331 Routers (R1, R2, R3)

Added two additional routers to simulate a remote site connected over an untrusted "service provider" router.

Implementation:

  • Configured IP addressing on multiple gigabit interfaces to create distinct internal and transit networks
  • Built a site-to-site IPsec VPN between R1 and R3, with R2 acting as an unaware transit router
  • Defined ACLs specifying "interesting traffic" between the 192.168.1.0/24 and 192.168.2.0/24 LANs

ISAKMP Phase 1 Configuration:

  • Encryption: AES-256
  • Authentication: Pre-shared key
  • DH Group 5 for key exchange

IPsec Phase 2:

  • Transform sets using ESP-AES-256 and ESP-SHA-HMAC for confidentiality and integrity
  • Applied crypto map to transit-facing interfaces on both routers

Verification:

  • Ping between hosts on each LAN through the VPN
  • Observed that transit router R2 simply forwards encrypted traffic with no payload visibility

4.4 Perimeter Security: ASA Firewall & DMZ

Tools Used: Cisco ASA 5506-X Firewall

Deployed a Cisco ASA 5506-X firewall between the internal LAN, DMZ, and outside network.

Security Zones:

  • Inside (100) – Trusted internal network
  • DMZ (50) – Semi-trusted public services segment
  • Outside (0) – Untrusted external network

Configuration:

  • Assigned IP addresses to each ASA interface and enabled them with no shutdown
  • Created network objects and NAT rules to allow inside and DMZ hosts to reach the outside network
  • Built ACLs to implement policy:
    • Permit ICMP for testing/monitoring
    • Permit HTTP (80) and HTTPS (443) outbound
    • Explicitly deny FTP (21) to demonstrate blocking of legacy protocols

4.5 Monitoring & Logging: Syslog, NTP, SNMP, and IPS

Tools Used: Syslog/NTP Server, SNMP, IOS IPS on Cisco 4331 Routers

Syslog & NTP Server:

  • Configured dedicated syslog & NTP server
  • Enabled syslog service and pointed routers/switches to this host as logging target
  • Configured NTP on devices to synchronize clocks for consistent timestamps

SNMP on R3:

  • Created public (read-only) and private (read-write) community strings
  • Used MIB Browser to query interface stats, uptime, and system info

IOS IPS (Intrusion Prevention System):

  • Created IPS directory in flash and pointed IPS config there
  • Enabled "basic" signature category and retired full "all" set for performance balance
  • Applied IPS rule to traffic on LAN-facing interface (ip ips rule 1 out)
  • Enabled log timestamps with millisecond granularity for alert correlation

4.6 DHCP, Time, and Address Management

Tools Used: Cisco ASA 5506-X, Cisco 3504 WLC

  • Configured DHCP scopes on the ASA firewall for inside and DMZ networks
  • Set up internal DHCP on the wireless LAN controller for wireless clients, aligned to same subnets
  • Verified automatic address assignment on all DHCP-enabled segments
  • Updated diagrams/documentation when static IPs changed

4.7 Wireless LAN Controller & Secure Wi-Fi

Tools Used: Cisco 3504 Wireless LAN Controller (WLC), Lightweight Access Points

Added a Cisco 3504 Wireless LAN Controller (WLC) and two lightweight APs to the inside network.

Configuration:

  • Converted laptops from wired NICs to wireless NICs and enabled DHCP
  • Configured management interface and connected via HTTPS from wired PC

WLANs Created:

  • Staff (employee) – WPA2-PSK with passphrase Cisco123
  • Guest – WEP with 40-bit static key (A1B2C3D4D5) to demonstrate weaker legacy configuration

Verification:

  • Laptops could join Staff and Guest SSIDs and obtain IP addresses via DHCP
  • Clients fail over between APs when one AP loses power, showing high availability
