Operational Security Flows

5.1 On-Ramp → Self-Custody (CEX Withdrawal)

Objective:

Buy on Coinbase/Gemini, withdraw to hardware wallet

Risk Controls:
  • Address book/whitelist verification
  • Test with dust amount first
  • Confirm full address (not just first/last chars)
  • Verify correct network (ERC-20 vs. Base)
Artifacts:

On-chain withdrawal transactions, whitelist documentation

5.2 Self-Custody → DEX Swap (Price/Slippage Discipline)

Objective:

Swap tokens on Uniswap/Jupiter with slippage caps

Risk Controls:
  • Verify token contract address
  • Check liquidity depth before swap
  • Simulate quote vs. execution price
  • MEV awareness (use limit orders/routers)
Artifacts:

Swap transactions, slippage logs, approval transactions

5.3 Provide Liquidity (LP on Toros)

Objective:

Supply LP, capture fees/APR, then exit

Risk Controls:
  • Document impermanent loss calculations
  • Verify realistic fee/APR claims
  • Set approval scope with spend limits
  • Test remove-liquidity flow before large deposits
Artifacts:

LP position tracking, fee earnings, IL calculations

5.4 Bridge / Chain Hop (Base/Arbitrum/Polygon)

Objective:

Move funds cross-chain to optimize fees or access new DeFi protocols

Risk Controls:
  • Use official bridge allow-list only
  • Verify RPC endpoint trust
  • Confirm destination chain address format
  • Post-bridge test spend to confirm receipt
Artifacts:

Bridge transaction hashes, cross-chain verification

5.5 Off-Ramp to Fiat (CEX Deposit & Withdrawal)

Objective:

Realize profits or rebalance portfolio to fiat

Risk Controls:
  • Correct memo/tag where applicable
  • Verify KYC/withdrawal limits
  • Confirm expected settlement time
  • Document for tax reporting
Artifacts:

Bank statements, CEX transaction history, tax reports

Threat Matrix & Defenses

Address Poisoning
Whitelists + full address verification + similarity alerts + test transactions
Fake Approvals
Hardware signing + approval-spend caps + regular revocation audits
Dusting Attacks
Ignore zero-value transfers + filter spam tokens in wallet UI
Fake Token Tickers
Verify contract address on Etherscan + check liquidity depth
Bridge Spoofing
Official bridge allow-list + RPC trust verification + community verification
Clipboard Malware
Manual address verification + QR code scanning + hardware wallet confirmation
Incident Response Playbook:

"If anomaly detected → freeze operations, revoke approvals, rotate RPC endpoints, rotate addresses, re-verify all destinations, document incident for analysis."

Lessons Learned & Security Takeaways

Top 5 Gotchas Encountered:

1. Wrong Network Selection

Nearly sent ERC-20 USDC to a Solana address. Caught by whitelist pre-check and network mismatch warning.

2. Blind Approval Requests

DEX approval for unlimited token spend. Now enforce approval caps and regular revocation audits.

3. Vanity Address Traps

Address poisoning attempt with near-identical prefix/suffix. Whitelisting protocol prevented loss.

4. Zero-Value Token Spam

Received fake tokens mimicking legitimate ones. Now filter spam tokens and verify contracts.

5. LP Impermanent Loss

Underestimated IL on volatile pairs. Now calculate IL before LP entry and monitor actively.

Mitigations That Worked:

  • Whitelists + Hardware Signing: Prevented address poisoning and phishing
  • Similarity Alerts: Custom scripts to flag near-match addresses
  • ERC-20 Zero-Value Filtering: Mental model to ignore spam tokens
  • Spend Limit Approvals: Never approve unlimited—always set caps
